# SCM_HW1_Elias

This repository was created as part of the Software Configuration Management (SWE442) assignment on
Version Control & Change Management. It shows the use of GitHub for
software configuration management, including repository setup, branching
strategies, feature development, and issue tracking.

## SCM Best Practices

Effective Software Configuration Management relies on several key practices:

- **Use meaningful commit messages:** Each commit should clearly describe what
  changed and why, making history easy to understand.
- **Branch for every feature or fix:** Never work directly on the main branch.
  Use feature branches to isolate work and reduce risk.
- **Review before merging:** Use pull requests to review changes before they
  reach the main branch, catching errors early.
- **Tag releases:** Use Git tags to mark stable release points, making it easy
  to roll back or reproduce a specific version.
- **Keep the repository clean:** Remove stale branches after merging and avoid
  committing generated files or secrets.
